通过Thread类

结果：

每次运行，可以看到顺序并不同，说明多线程里，那个线程先执行、什么时候执行 均取决于CPU资源的调度。

在注释中m.start();是无法多次调用该方法实现多线程共享同一个对象资源的，因为：start()方法是启动/创建一个新线程，新线程会执行相应的run方法，其不能被同一个对象重复调用。
